  <action>
    **src/lib/fmp/types.ts** — TypeScript interfaces for all FMP API responses we'll use:
    - FMPProfile (name, price, mktCap, beta, sector, range, image, etc.)
    - FMPQuote (price, change, changesPercentage, dayLow, dayHigh, volume, yearLow, yearHigh, etc.)
    - FMPKeyMetrics (peRatioTTM, pegRatioTTM, priceToSalesRatioTTM, evToEbitda, roeTTM, roicTTM, freeCashFlowYieldTTM, debtEquityRatioTTM, etc.)
    - FMPRatios (operatingProfitMarginTTM, netProfitMarginTTM, returnOnEquityTTM, returnOnAssetsTTM, currentRatioTTM, etc.)
    - FMPGrowth (revenueGrowth, epsgrowth, freeCashFlowGrowth, etc.)
    - FMPIncomeStatement (date, revenue, netIncome, operatingIncome, etc.)
    - FMPCashFlow (date, operatingCashFlow, freeCashFlow, etc.)
    - FMPBalanceSheet (date, longTermDebt, totalCurrentAssets, totalCurrentLiabilities, etc.)
    - FMPDCF (dcf, price, date)
    - FMPEstimate (estimatedRevenueAvg, estimatedEpsAvg, etc.)
    - FMPRecommendation (analystRatingsStrongBuy, analystRatingsBuy, analystRatingsHold, analystRatingsSell, analystRatingsStrongSell, etc.)
    - FMPHistoricalPrice (date, open, high, low, close, volume)
    - FMPTechnicalIndicator (date, close, [indicator-specific field like ema, sma, rsi])
    - FMPSectorPerformance (sector, changesPercentage)
    - FMPMarketMover (symbol, name, change, price, changesPercentage)
    - FMPEconomicEvent (date, event, estimate, actual, previous, impact)
    - FMPTreasuryRate (date, month1, month2, month3, month6, year1, year2, year5, year10, year20, year30)
    - FMPSectorPE (date, sector, pe)

    Use `interface` not `type` for all API shapes. Fields should be typed as `number | null` where FMP may return null.

    **src/lib/fmp/endpoints.ts** — Endpoint URL builders:
    - Define BASE_V3 = "https://financialmodelingprep.com/api/v3"
    - Define BASE_V4 = "https://financialmodelingprep.com/api/v4"
    - Define BASE_STABLE = "https://financialmodelingprep.com/stable"
    - Export functions that take (ticker, apiKey) and return full URL for each endpoint
    - Group into: marketEndpoints (indices, VIX, sectors, movers, calendar, treasury, SPX technicals) and tickerEndpoints (profile, quote, keyMetrics, ratios, growth, income, cashflow, balance, dcf, estimates, recommendations, history, ema20, sma50, sma150, sma200, rsi, sectorPE)
    - Include today's date computation for sectorPE endpoint

    **src/lib/fmp/client.ts** — The core FMP client:
    - `createFMPClient(apiKey: string)` factory function that returns an object with methods
    - Internal state: requestCount (number), cache (Map<string, { data: unknown; timestamp: number }>)
    - `getRequestCount()`: returns current count
    - `resetRequestCount()`: resets to 0
    - `fetch<T>(url: string, cacheKey: string, ttl?: number)`: core fetch with caching
      - Check cache first (default TTL: 5 minutes)
      - If cache miss: increment requestCount, fetch from URL, parse JSON, store in cache, return typed data
      - On error: return null (do NOT throw — individual endpoint failures should not break the whole dashboard)
      - Log errors to console with endpoint context
    - `fetchMarketData()`: calls all market endpoints in parallel with Promise.allSettled, returns typed market data object
    - `fetchTickerData(ticker: string)`: calls all ticker endpoints in parallel with Promise.allSettled, returns typed ticker data object
    - `clearCache()`: clears all cached data (for refresh button)

    Do NOT use localStorage for caching — in-memory only (session-scoped).
    Do NOT use React Query or SWR — keep it simple with native fetch and manual cache.
    Use `Promise.allSettled` (not `Promise.all`) so one failing endpoint doesn't break everything.
  </action>
